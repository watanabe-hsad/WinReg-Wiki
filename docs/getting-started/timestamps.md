# 时间戳语义

注册表 key 有 LastWrite 时间，但 value 本身通常没有独立时间戳。这一点是注册表取证最容易误判的地方。

## 常见结论边界

| 观察 | 较稳妥的说法 | 不应直接说成 |
|---|---|---|
| 某 key 的 LastWrite 更新 | 该 key 或其直接内容在该时间附近发生变化 | 某个具体 value 一定在该时间创建 |
| Run key 存在启动项 | 该程序被配置为登录启动 | 该程序一定已经执行 |
| UserAssist 计数增加 | 用户通过 Explorer 相关交互触发过程序 | 程序一定由用户手动双击启动 |
| USBSTOR 存在设备序列号 | 系统识别过该 USB 存储设备 | 文件一定被拷贝 |

## 建议写法

报告里可以使用更严谨的表达：

```text
该注册表项显示系统曾记录一个与该设备相关的枚举信息。结合 MountedDevices、
事件日志和文件系统时间线，可进一步判断设备连接时间和盘符分配情况。
```
